Để áp dụng bộ tài liệu này cho một dự án mới, BA nên trả lời theo thứ tự:

1. Learning container của dự án là gì?
2. Learning item là gì?
3. Session type nào cần có?
4. Mode nào phù hợp với đặc thù kiến thức?
5. Outcome model là gì?
6. Retry model là gì?
7. Learning state model là gì?
8. Reporting và KPI cần gì?

## Conclusion

Một study feature generic không nên được nhìn như “một màn hình làm bài”.

Nó nên được nhìn như một `business engine` với 8 khái niệm lõi:

1. learning container
2. learning item
3. session type
4. mode plan
5. action-state contract
6. outcome model
7. retry model
8. long-term learning state

Nếu 8 khái niệm này được đặc tả rõ, phần còn lại như UI, API, database hay framework chỉ còn là bài toán triển khai.
